    why setting up static internet protocol(IP) is beneficial?

    • When there is lot of devices on your network—computers, NAS devices, and Raspberry Pis—it can be annoying to constantly look up their IP addresses when you need to access them remotely. Make your life easier with a set of unchanging, static IP addresses for those machines.

    • Usually router assigns IP addresses to devices on your network using DHCP, or Dynamic Host Configuration Protocol. As you connect new devices to the network, they'll be assigned the next IP address in the pool, and if a device hasn't connected in a few days, its IP address will "expire" so it can be assigned to something else.

    • For everyday use, this is perfectly fine, and you'll never even notice it happening in the background. But if you regularly SSH into your Raspberry Pi, use Wake-on-LAN to turn your computer on from across the house, or perform other advanced networking tasks, DHCP can become an annoyance. It's hard to remember which IP address is assigned to which device, and if they ever expire, you have to look it up all over again. That's where a static IP address comes in handy.
